## Task
Update `concept/09_open_questions.md` with unresolved assumptions, questions or postponed decisions.

## Rules
- Do not use open questions as a dumping ground for solved decisions.
- If a question is resolved, mark it as resolved or move the resulting decision into an ADR/reference/concept file.
- Classify open questions by area: product, security, npm, CLI, UI, usage analysis, implementation.
- Keep entries short and actionable.

## Return
1. Questions added
2. Questions resolved
3. Questions that require ADRs
4. Recommended owner/skill for each open question
